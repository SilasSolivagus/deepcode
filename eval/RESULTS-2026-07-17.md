# deepcode 能力评测报告（2026-07-17）

> 用可复现的自建 eval harness（`eval/`）+ 与顶级闭源 agent 同题对打，量化 deepcode（DeepSeek/GLM 终端编码 agent）在真实任务上的能力与**成本-可靠性 Pareto**。

## 方法论（对齐行业最佳实践）

- **防污染**：全部场景自建、不在任何公开数据集里，模型无法背题。
- **程序化判分**：结果客观可判（测试过 / 数值对 / 文件有效 / 命中预埋异常），不靠主观打分。
- **pass^N 可靠性**：每格跑 3 个 seed；`pass^3` = 3 次全过 = 可靠（照 τ-bench 的可靠性理念，抓 flaky）。
- **多模型 + 成本/延迟一等指标**：4 个国产模型横评，记录每格平均成本（CNY）与耗时。
- **能力隔离**：harness 与模型分离——同一 harness 换模型，度量的是模型；harness 行为本身单独评。

## 一、pass^3 矩阵（5 场景 × 4 模型 × 3 seed）

| 场景 | deepseek-v4-pro | deepseek-v4-flash | glm-5-turbo | glm-5.2 |
|---|---|---|---|---|
| bugfix（定位修 bug） | 3/3★ ¥.022 9s | 3/3★ ¥.006 6s | 3/3★ ¥.010 14s | 3/3★ ¥.089 35s |
| refactor（跨文件抽公共函数） | 3/3★ ¥.027 13s | 3/3★ ¥.009 9s | 3/3★ ¥.008 13s | 3/3★ ¥.088 26s |
| recovery（测试失败自愈） | 3/3★ ¥.025 11s | 3/3★ ¥.009 8s | 3/3★ ¥.011 14s | 3/3★ ¥.104 25s |
| **evaluator（复杂 parser）** | **3/3★** ¥.066 59s | **1/3** ¥.022 41s | **0/3** ¥.062 108s | **3/3★** ¥.816 145s |
| log-analysis（非编程·日志分析） | 3/3★ ¥.087 52s | 3/3★ ¥.034 47s | 3/3★ ¥.027 56s | 3/3★ ¥.513 68s |
| **总计** | **15/15 · 5/5可靠 · ¥0.68** | 13/15 · 4/5 · ¥0.24 | 12/15 · 4/5 · ¥0.35 | 15/15 · 5/5 · ¥4.83 |

★ = pass^3（3 次全过）。

**关键发现**：
- **复杂编程（表达式求值器）是唯一分水岭**——其余 4 个场景四模型全 3/3★。
- **pass^N 抓出 flaky**：deepseek-flash 的 evaluator 是 **1/3**（2 次栽在小数解析、1 次蒙过）——单次跑会误判它 OK，多 seed 才照出不可靠。
- glm-5-turbo evaluator **0/3 稳定失败**（把 `1++2` 判为 3，一元正号）；glm-5.2（智能档）**3/3 过**——档位差在难题显形。

## 二、成本-可靠性 Pareto（差异化结论）

| 模型 | 可靠场景(pass^3) | 总花费 | 定位 |
|---|---|---|---|
| **deepseek-v4-pro** | **5/5** | **¥0.68** | **Pareto 赢家：与最贵的 glm-5.2 同样可靠，成本仅 1/7** |
| deepseek-v4-flash | 4/5 | ¥0.24 | 极致性价比，只在最难 parser 边界 flaky |
| glm-5-turbo | 4/5 | ¥0.35 | 中庸，evaluator 失败 + 负载下变慢 |
| glm-5.2 | 5/5 | ¥4.83 | 最可靠但最贵最慢，被 deepseek-pro 帕累托支配 |

> **一句话**：deepseek 系全面碾压 glm 系的性价比；**deepseek-v4-pro = 顶级可靠 + 便宜 7×**。deepcode 的差异化＝「**国产便宜模型上做扎实、可控的 harness = 顶级可靠、几分之一成本**」，此处首次有 pass^3 数据背书。

## 三、与顶级闭源 agent 同题对打（4 个复杂 + 多样任务）

deepcode（deepseek-v4-pro）vs 对照 agent，同 prompt 同环境：

| 任务 | 结果 |
|---|---|
| Office 三件套（docx+xlsx+pptx） | 两侧都产出有效文件、结构达标；打平（CC 用 Excel 活公式更地道，deepcode 表列更丰富） |
| 深度研究（向量数据库对比，联网） | 两侧都覆盖 4 库 4 维、带来源；打平 |
| 数据分析（预埋异常日志） | 两侧都精确揪出两个预埋异常、数字分毫不差；打平 |
| 复杂编程（求值器） | 两侧测试全过 + 9 刁钻输入全对；打平 |

**结论**：在这一类「编排 + 工具 + 结构化输出」的真实复杂任务上，**deepcode + deepseek-v4-pro 全面追平 Claude**。诚实边界：最烧脑的深推理（微妙算法 bug / 超大代码库 / 深架构）上 Claude 可能仍领先——那主要是模型能力，非 harness。

## 四、harness 骨架评（B/C/D/F 简单场景 + 4 复杂任务）

deepcode 的核心 loop 与工具编排在所有任务上零故障：Read/Grep/Edit/Bash/Glob、改完主动重跑验证、模糊需求先问清不瞎猜。**换厂换档都不塌**，证明骨架扎实、与模型解耦。

## 五、公开榜单进度（Terminal-Bench）

deepcode 的 Terminal-Bench 适配器（`AbstractInstalledAgent`，照 qwen-code/claude-code 模式）**已写好、可移植**。本地跑受阻——**`registry.terminal-bench.ai` 从本机网络持续不可达（HTTP 000）**，数据集下不下来、harness 挂死（非 deepcode 或适配器问题）。→ 可比榜单数字需在**国际网络通畅的干净环境**（云 CI）跑；适配器已 turnkey。SWE-bench（数据托管 HuggingFace，本机可达）作为替代/补充评估中。

## 复现

```bash
node eval/run.mjs --models deepseek-v4-pro,deepseek-v4-flash,glm-5-turbo,glm-5.2 --seeds 3
```
